SERAC Compiler Roadmap (SERAC → C → GCC)

MVP:
    Namespaces
    Components (struct-like types)
    Primitive types (i32, bool, etc.)
    Pointers
    ref, mut ref
    Functions
    Methods (lowered to functions)
    Control flow (if, while, maybe for)
    Basic expressions (binary ops, calls, field access)
    ret
    Basic visibility (pub)

2) Build Lexer
   - Tokenize source files (keywords, identifiers, literals, symbols).
   - Track file/line/column spans for diagnostics.
   Deliverable: Tool that reads SERAC source and prints token stream with correct positions.

3) Build Parser → AST
   - Parse tokens into an Abstract Syntax Tree.
   - Preserve spans on all nodes.
   Deliverable: Tool that parses full SERAC files and produces a valid AST (no codegen yet).

4) Name Resolution (Symbol Binding)
   - Build symbol tables for namespaces, types, functions, templates.
   - Resolve identifiers to definitions.
   - Enforce visibility rules (pub/private).
   Deliverable: Parsed programs with all names resolved or cleanly rejected with errors.

5) Core Type System (Non-Template First)
   - Implement type representations (primitive, pointer, ref, component, function types).
   - Type-check expressions and statements.
   - Enforce mut/ref rules and parameter passing rules.
   Deliverable: Non-generic SERAC programs fully type-check and reject invalid code.

6) Template System + Monomorphization
   - Implement template instantiation for types and functions.
   - Generate concrete instances per unique type argument set.
   Deliverable: Generic SERAC programs compile into concrete, fully instantiated internal representations.

7) Implement comptime + @match
   - Evaluate comptime expressions during compilation.
   - Implement type-pattern matching with variable binding.
   - Ensure only selected comptime branch is semantically analyzed.
   Deliverable: Templates using comptime if and @match behave correctly and deterministically.

8) Semantic Lowering Pass
   - Desugar method syntax into free functions with explicit self.
   - Normalize control flow and simplify expressions.
   - Insert required destroy/move/init hooks if applicable.
   Deliverable: Clean, simplified intermediate representation ready for code generation.

9) C Code Generation Backend
   - Define name mangling scheme.
   - Emit C structs, functions, globals.
   - Map SERAC refs/mut refs to C pointer conventions.
   - Insert #line directives for accurate error mapping.
   Deliverable: SERAC source → generated C code that compiles with GCC.

10) End-to-End Compilation Driver
    - CLI tool: seracc input.sr → output executable.
    - Invoke GCC automatically.
    Deliverable: Working SERAC compiler that produces runnable binaries.

11) Runtime + Core Library Support
    - Provide minimal C runtime helpers if needed.
    - Implement core utilities (memory, assertions, etc.).
    Deliverable: Non-trivial example programs (like your container + iterator examples) run correctly.

12) Hard Semantic Features
    - Enforce @needs_destroy / relocation rules.
    - Implement restrictions like forbidden copy (=) for flagged types.
    Deliverable: Advanced type semantics work as specified.

13) Diagnostics & Stability
    - Improve template error messages.
    - Add better error reporting with source spans.
    - Add regression tests.
    Deliverable: Usable compiler with clear diagnostics and stable behavior.
